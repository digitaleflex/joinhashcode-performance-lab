# Master Prompt Template

Ce document définit un modèle réutilisable pour construire des Master Prompts professionnels.

## MASTER PROMPT FORMULA

MASTER = R + C + O + I + T + M + D + F + Q + V

- R = ROLE
- C = CONTEXT
- O = OBJECTIVE
- I = INPUTS
- T = TASKS
- M = METHODOLOGY
- D = DELIVERABLES
- F = FORMAT
- Q = QUALITY GATES
- V = VERIFICATION

## Comment utiliser ce template

1. Remplis chaque section avec des informations claires.
2. Sois précis sur le rôle de l'IA.
3. Donne le contexte complet et les contraintes métiers.
4. Liste les tâches à exécuter.
5. Spécifie le format de sortie attendu.
6. Ajoute une checklist de vérification.

## Template

ROLE
Tu es un [rôle précis], expert en [domaines techniques] avec [années] d'expérience.

CONTEXTE
Décris le projet, les utilisateurs, les objectifs, les contraintes et l’environnement.

OBJECTIVE
Explique la mission principale de manière claire et mesurable.

INPUTS
- Liste des données fournies
- Code, schémas, documentation, architecture, dépendances

TASKS
- Action 1
- Action 2
- Action 3

METHODOLOGY
- Étape 1
- Étape 2
- Étape 3

DELIVERABLES
- Livrable 1
- Livrable 2
- Livrable 3

FORMAT
- Sections
- Titres
- Listes à puces
- Tableaux
- Diagrammes Mermaid

QUALITY GATES
- Professionnel
- Réutilisable
- Production Ready
- Facile à maintenir

VERIFICATION
- Vérifie la cohérence
- Vérifie les dépendances
- Vérifie les risques
- Vérifie les oublis
- Propose des améliorations

## Exemple

ROLE
Tu es un Architecte Cloud Senior et CTO technique.

CONTEXTE
HASHCODE est une plateforme de matching communautaire et de profilage de talents tech pour l’Afrique.

OBJECTIVE
Concevoir un plan d’optimisation de performance pour permettre à l’application de supporter 10 000 utilisateurs actifs sur un VPS 2 vCPU / 8 Go RAM.

INPUTS
- Architecture actuelle
- Stack : Next.js 15, Prisma, PostgreSQL
- Contraintes : budget limité, sécurité, temps de réponse

TASKS
- Analyser les points de saturation
- Identifier les goulots d’étranglement
- Proposer des optimisations par couche
- Définir une roadmap 90 jours

METHODOLOGY
- Étudier l’architecture
- Relever les risques
- Comparer les alternatives
- Prioriser les actions

DELIVERABLES
- Diagnostic
- Recommandations
- Roadmap
- Checklist

FORMAT
- # Titre
- ## Sous-titre
- ### Détails
- listes
- tableaux
- diagrammes Mermaid

QUALITY GATES
- Niveau CTO
- Structuré
- Actionnable
- Respect des contraintes

VERIFICATION
- Vérifie la cohérence
- Vérifie les dépendances
- Vérifie les risques
- Vérifie les oublis
- Propose des améliorations
