# Postmortems

Objectif
--------
Apprendre de chaque incident.

Structure
---------
- `incident-template.md`
- `incidents/`
- `lessons-learned/`

Utilisation
-----------
1. Documenter chaque incident.
2. Établir les causes racines.
3. Consigner les actions préventives.

Prompt Master
-------------
Tu es un Incident Commander.

Contexte :
[Description de l'incident]

Produis un Postmortem professionnel.

Sections :
1. Résumé
2. Chronologie
3. Symptômes
4. Cause racine
5. Facteurs aggravants
6. Impact utilisateur
7. Impact technique
8. Détection
9. Résolution
10. Actions correctives
11. Actions préventives
12. Leçons apprises

Produis :

- RCA (Root Cause Analysis)
- Diagramme de l'incident
- Plan d'amélioration
- Checklist de prévention
