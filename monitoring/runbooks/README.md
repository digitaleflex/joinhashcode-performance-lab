# Runbooks

## Objectif
Fournir des procédures claires en cas d'incident.

## Contenu recommandé
- Résolution d'alerte de latence
- Résolution d'alerte d'erreur
- Résolution d'alerte de saturation CPU/RAM
- Vérification PostgreSQL
- Gestion des redémarrages Docker

## Utilisation
1. Documenter les étapes de diagnostic.
2. Mentionner les outils à utiliser.
3. Lister les responsables et les contacts.
